# Streamlit Tree Visualizer

Interactive web-based visualizer for exploring hierarchical document structure generated by the RAG ingestion pipeline.

## Overview

The Streamlit Tree Visualizer provides a user-friendly interface to:
- Browse document hierarchy (chapters → sections → subsections)
- View full text content of any node
- Search across all nodes by keyword
- Understand how documents are indexed for retrieval

## Features

### 1. Hierarchical Tree View
- **Top-to-bottom tree layout** with parent-child relationships
- **Color-coded levels** (L1-L6) for visual depth indication
- **Expand/collapse** branches with 📂 button
- **Node metadata**: level, child count, text length

### 2. Node Content Viewer
- **Full text display** of any node with proper formatting
- **Breadcrumb navigation** showing complete path (e.g., "Document > Chapter 1 > Section 1.1")
- **Metadata panel**: page numbers, node type, character/token counts
- **Vietnamese text support** with proper encoding

### 3. Search Functionality
- **Full-text search** across titles and content
- **Context preview** with highlighted matches
- **Direct navigation** to view full node from search results
- **Case-insensitive** search with preview snippets

## Access

### Starting the Visualizer

The visualizer runs as a Docker service:

```bash
# Start all services (including visualizer)
docker compose up --build

# Or start only visualizer after other services are running
docker compose up visualizer
```

### Web Interface

Once running, access the visualizer at:
- **URL**: http://localhost:8501
- **No authentication required** (internal tool)

## API Endpoints

The visualizer uses these backend API endpoints (all require JWT authentication):

### 1. Get Document Tree
```http
GET /api/v1/tree/{document_id}
Authorization: Bearer <token>
```

**Response:**
```json
{
  "document_id": "uuid",
  "document_title": "Document.pdf",
  "total_nodes": 45,
  "max_depth": 3,
  "nodes": [
    {
      "node_id": "uuid",
      "title": "Chapter 1",
      "level": 1,
      "breadcrumb": "Document > Chapter 1",
      "parent_id": null,
      "child_count": 5,
      "text_length": 2500,
      "page_number": 1
    }
  ]
}
```

### 2. Get Node Details
```http
GET /api/v1/tree/{document_id}/nodes/{node_id}
Authorization: Bearer <token>
```

**Response:**
```json
{
  "node_id": "uuid",
  "title": "Section 1.1",
  "level": 2,
  "breadcrumb": "Document > Chapter 1 > Section 1.1",
  "text": "Full context/text content...",
  "metadata": {
    "page_number": 5,
    "node_type": "section",
    "order": 1,
    "char_count": 2500,
    "token_count": 500
  }
}
```

### 3. Search Nodes
```http
GET /api/v1/tree/{document_id}/search?query=keyword
Authorization: Bearer <token>
```

**Response:**
```json
{
  "results": [
    {
      "node_id": "uuid",
      "title": "Matching Section",
      "preview": "...context around match...",
      "highlight": "matched text"
    }
  ]
}
```

## Usage Guide

### Step 1: Get Document ID
You need a document UUID to visualize:

```bash
# List all documents
curl -X GET http://localhost:8000/api/v1/documents \
  -H "Authorization: Bearer <token>"

# Or from API response after upload
```

### Step 2: Open Visualizer
1. Navigate to http://localhost:8501
2. Enter Document ID in the sidebar
3. Click "🔄 Load Tree"

### Step 3: Explore Document
- **Click node titles** to view full content
- **Use 📂 button** to expand/collapse sections
- **Switch to "Search" tab** to find specific text
- **View metadata** by expanding the metadata panel

## Configuration

### Environment Variables
The visualizer uses these environment variables (set in `.env`):

```bash
# API connection
API_BASE_URL=http://api:8000
API_V1_PREFIX=/api/v1

# Streamlit settings
STREAMLIT_SERVER_PORT=8501
STREAMLIT_SERVER_ADDRESS=0.0.0.0
STREAMLIT_SERVER_HEADLESS=true
```

### Docker Service
The visualizer is defined in `docker-compose.yml`:

```yaml
visualizer:
  image: chatbot-rag-app:local
  command: >
    python -m streamlit run app/view/visualizer.py
    --server.port=8501
    --server.address=0.0.0.0
    --server.headless=true
    --server.enableCORS=false
    --server.enableXsrfProtection=false
  ports:
    - "8501:8501"
  depends_on:
    api:
      condition: service_healthy
```

## Troubleshooting

### Issue: Cannot connect to API
**Symptom**: Visualizer shows "Error fetching tree"

**Solution:**
1. Check if API service is running: `docker compose ps api`
2. Check API health: `curl http://localhost:8000/api/v1/health`
3. Verify API_BASE_URL in visualizer matches API service

### Issue: "Document not found"
**Symptom**: API returns 404 for document ID

**Solution:**
1. Verify document exists: `curl http://localhost:8000/api/v1/documents -H "Authorization: Bearer <token>"`
2. Check document status is "completed" (not "pending" or "processing")
3. Ensure correct document UUID (copy from API response)

### Issue: No nodes in tree
**Symptom**: Tree loads but shows 0 nodes

**Solution:**
1. Check ingestion completed successfully: `curl http://localhost:8000/api/v1/status/<task_id>`
2. Verify Qdrant has vectors:
   ```bash
   curl http://localhost:6333/collections/ragbot
   ```
3. Check worker logs: `docker compose logs -f worker`

### Issue: Vietnamese text displays incorrectly
**Symptom**: Garbled characters or encoding errors

**Solution:**
1. Ensure UTF-8 encoding in browser
2. Check database charset is UTF-8
3. Verify original PDF has proper text encoding

## Architecture

### Components
1. **Streamlit App** (`app/view/visualizer.py`)
   - Frontend UI with interactive widgets
   - HTTP client for API calls
   - Tree rendering logic

2. **Tree API** (`app/api/routes/tree.py`)
   - FastAPI endpoints
   - Qdrant vector store queries
   - Tree structure builder

3. **Qdrant Vector Store**
   - Stores hierarchical nodes with metadata
   - Fast retrieval by document_id
   - Full-text search capability

### Data Flow
```
Streamlit UI → API Request → Tree API → Qdrant Query → Tree JSON → UI Render
```

### Security
- API endpoints require JWT authentication
- Visualizer runs in headless mode (no code execution)
- CORS disabled for local development
- Rate limiting applies to API calls

## Performance

### Expected Response Times
- **Small document** (<10 nodes): <100ms
- **Medium document** (10-50 nodes): <500ms
- **Large document** (50-200 nodes): <2s
- **Search query**: <500ms

### Optimization Tips
1. **Limit tree depth** for very large documents
2. **Use pagination** if document has >1000 nodes
3. **Cache tree data** in Streamlit session state
4. **Enable Qdrant indexing** for faster searches
