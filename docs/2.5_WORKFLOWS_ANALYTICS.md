# 2.5 — Analytics Data Flow

## Token Tracking Architecture

Every AI model call is logged into `ai_model_usage` table via fire-and-forget Celery tasks:

```
OpenAILike.astream_chat() / acomplete()
       ↓
provider.last_usage (populated after response)
       ↓
track_usage() reads last_usage, dispatches Celery task
       ↓
log_model_usage_task (fire-and-forget)
       ↓
INSERT INTO ai_model_usage (model_type=llm)

TEIRerankerPostprocessor / NvidiaRerankerPostprocessor
       ↓
_track_reranker_usage() → log_model_usage_task (model_type=reranker)
```

### Tracked Model Types

| Model Type | Source | Purpose |
|------------|--------|---------|
| `llm` | `OpenAILike` via `Settings.llm` | Main QA response |
| `reranker` | `TEIRerankerPostprocessor` / `NvidiaRerankerPostprocessor` | Document reranking |

### ai_model_usage Table

| Column | Source |
|--------|--------|
| `model_name` | Model identifier (e.g., `nvidia/llama-nemotron-rerank-1b-v2`) |
| `model_type` | `llm`, `embedding`, or `reranker` |
| `prompt_tokens` | Input tokens (estimated for embedding/reranker: chars/3) |
| `completion_tokens` | Output tokens (0 for reranker) |
| `latency_ms` | Response time in milliseconds |
| `endpoint` | API endpoint identifier |
| `cost_usd` | Estimated from provider rates |
| `user_id`, `session_id`, `message_id` | Optional context |

## Analytics Aggregation

```
ai_model_usage (all model types)
       ↓
AnalyticsRepository:
  get_totals() → SUM tokens, cost by period
  get_model_type_stats() → GROUP BY model_type (llm, embedding, reranker)
  get_daily_stats() → GROUP BY date
  get_daily_stats_by_model_type() → GROUP BY date + model_type
       ↓
GET /analytics/stats?days=N → system-wide (admin) or personal (member)
```

## Analytics Endpoints

| Endpoint | Access | Returns |
|----------|--------|---------|
| `GET /analytics/stats?days=1` | Admin/Member | Today's stats |
| `GET /analytics/stats?days=7` | Admin/Member | Last 7 days |
| `GET /analytics/stats?days=30` | Admin/Member | Last 30 days (default) |
