# 2.5 — Analytics Data Flow

## Overview

Tokens stored per ChatMessage → aggregated by SQL query → endpoint. Admin sees system-wide, members see own stats. Pricing configurable via `AI_INPUT_COST_PER_1M`.

## Data Flow

```
ChatMessage (tokens_in, tokens_out, latency_ms, model_used, estimated_cost_usd)
       ↓
SQL Aggregation (GROUP BY user_id, date)
       ↓
GET /analytics/stats
       ↓
Admin → System-wide stats
Member → Own stats only
```

## Cost Tracking

| Field | Description |
|-------|-------------|
| `tokens_in` | Prompt tokens sent to LLM |
| `tokens_out` | Completion tokens received |
| `latency_ms` | Time to generate response |
| `model_used` | Which AI model (e.g., gemma-4) |
| `estimated_cost_usd` | Calculated from `AI_INPUT_COST_PER_1M` + `AI_OUTPUT_COST_PER_1M` |

## Pricing Configuration

```bash
AI_INPUT_COST_PER_1M=0.0   # Free tier default
AI_OUTPUT_COST_PER_1M=0.0
```

## Analytics Endpoints

| Endpoint | Access | Returns |
|----------|--------|---------|
| `GET /analytics/stats` | Admin | System-wide token usage, cost, sessions |
| `GET /analytics/stats` | Member | Own usage only |
