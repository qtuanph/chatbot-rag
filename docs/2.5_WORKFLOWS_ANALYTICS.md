# 2.5 — Analytics Workflow

Analytics hiện tại tập trung vào **usage model** chứ không còn dựa trên persisted chat session.

## Mục tiêu

- theo dõi usage theo tenant
- xem tenant nào tốn nhiều tài nguyên hơn
- tính quota / rate limit
- theo dõi latency và token

## Nguồn dữ liệu chính

`ai_model_usage`

## Những loại usage được ghi

| `model_type` | Ý nghĩa |
|---|---|
| `llm` | gọi model sinh câu trả lời |
| `embedding` | gọi model embedding |
| `reranker` | gọi model rerank |

## Dữ liệu quan trọng

| Cột | Ý nghĩa |
|---|---|
| `tenant_id` | tenant sở hữu usage |
| `user_id` | user nội bộ nếu có |
| `prompt_tokens` | token đầu vào |
| `completion_tokens` | token đầu ra |
| `total_tokens` | tổng token |
| `cost_micros_vnd` | tiền lưu dạng integer micro-VND |
| `latency_ms` | độ trễ |
| `endpoint` | route/luồng gọi |

## Quy tắc tiền tệ

- chuẩn tiền tệ hiện tại: `VND`
- không lưu `float`
- dùng `cost_micros_vnd`

## Endpoint chính

| Endpoint | Mục đích |
|---|---|
| `GET /analytics/stats` | thống kê chung |
| `GET /analytics/me/usage` | usage của user hiện tại |
| `GET /admin/users/usage` | thống kê theo user |
| `GET /admin/users/{user_id}/usage` | chi tiết user |
| `GET /admin/tenants/usage` | thống kê theo tenant |
| `DELETE /analytics/stats` | reset usage dev |

## Điều đã bỏ

Không còn xem:

- `total_sessions`
- analytics dựa trên `chat_sessions`
- analytics dựa trên persisted transcript

## Lưu ý thực tế

Embedding và reranker có thể không luôn có số token “đẹp” như LLM.

Vì vậy UI cần chấp nhận:

- `completion_tokens = 0`
- token ước tính cho embedding/reranker

Đó không hẳn là lỗi.
