# 2.5 — Analytics Workflow

Analytics hiện tại tập trung vào **usage model** và **feedback chất lượng**, không còn dựa trên persisted chat session.

## Mục tiêu

- theo dõi usage theo tenant
- xem tenant nào tốn nhiều tài nguyên hơn
- tính quota / rate limit
- theo dõi latency, token và cost
- theo dõi like / dislike để cải thiện chất lượng

## Nguồn dữ liệu chính

- `ai_model_usage`
- `chat_feedback`

## Những loại usage được ghi

| `model_type` | Ý nghĩa |
|---|---|
| `llm` | gọi model sinh câu trả lời |
| `embedding` | gọi model embedding |
| `reranker` | gọi model rerank |

## Dữ liệu usage quan trọng

| Cột | Ý nghĩa |
|---|---|
| `tenant_id` | tenant sở hữu usage |
| `user_id` | user nội bộ nếu có |
| `prompt_tokens` | token đầu vào |
| `completion_tokens` | token đầu ra |
| `total_tokens` | tổng token |
| `cost_micros_vnd` | tiền lưu dạng integer micro-VND |
| `latency_ms` | độ trễ |
| `endpoint` | route hoặc flow đã gọi |

## Dữ liệu feedback

Feedback chat lưu:

- `like` / `dislike`
- `query_text`
- `assistant_answer`
- `document_ids`
- `section_ids`
- `citations`
- `llm_model`, `embedding_model`, `reranker_model`

## Endpoint chính

| Endpoint | Mục đích |
|---|---|
| `GET /analytics/stats` | thống kê chung |
| `GET /analytics/me/usage` | usage của user hiện tại |
| `GET /admin/users/usage` | thống kê theo user |
| `GET /admin/users/{user_id}/usage` | chi tiết user |
| `GET /admin/tenants/usage` | thống kê theo tenant |
| `DELETE /analytics/stats` | reset usage dev |

`GET /analytics/stats` hiện cũng trả `feedback_summary` để UI hiển thị:

- tổng like / dislike
- dislike rate
- top tài liệu bị dislike
- top section bị dislike

## Quy tắc tiền tệ

- chuẩn tiền tệ là `VND`
- không lưu `float`
- dùng `cost_micros_vnd`

## Những gì đã bỏ

- `total_sessions`
- analytics dựa trên `chat_sessions`
- analytics dựa trên persisted transcript

## Lưu ý thực tế

Embedding và reranker có thể không luôn có token đẹp như LLM.

UI phải chấp nhận:

- `completion_tokens = 0`
- token ước tính cho embedding hoặc reranker
