## Token Tracking Architecture

Every AI model call is logged into `ai_model_usage` table via fire-and-forget Celery tasks:

```
OpenAILike.astream_chat() / acomplete()
       â†“

       â†“

       â†“

       â†“

TEIRerankerPostprocessor / NvidiaRerankerPostprocessor
       â†“

```

### Tracked Model Types

| Model Type | Source | Purpose |
|------------|--------|---------|
| `llm` | `OpenAILike` via `Settings.llm` | Main QA response |
| `reranker` | `TEIRerankerPostprocessor` / `NvidiaRerankerPostprocessor` | Document reranking |

### ai_model_usage Table

| Column | Source |
|--------|--------|
| `model_name` | Model identifier (e.g., `nvidia/llama-nemotron-rerank-1b-v2`) |
| `model_type` | `llm`, `embedding`, or `reranker` |
| `prompt_tokens` | Input tokens (estimated for embedding/reranker: chars/3) |
| `completion_tokens` | Output tokens (0 for reranker) |
| `latency_ms` | Response time in milliseconds |
| `endpoint` | API endpoint identifier |
| `cost_usd` | Estimated from provider rates |
| `user_id`, `session_id`, `message_id` | Optional context |

### Admin User Usage Detail (New)

`GET /admin/users/{user_id}/usage` returns `window_30d` with:

- `tokens_in`, `tokens_out`, `total_tokens`
- `estimated_cost_usd`
- `daily` token rows per date
- `by_model_type` for `llm`, `embedding`, `reranker`
