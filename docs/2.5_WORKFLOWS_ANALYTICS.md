# 2.5 — Analytics Data Flow

## Token Tracking Architecture

Every AI model call is logged into `ai_model_usage` table via `track_usage()`:

```
AIProxyBridge.chat() / chat_stream()
       ↓
provider.last_usage (populated after response)
       ↓
track_usage() reads last_usage, dispatches Celery task
       ↓
log_model_usage_task (fire-and-forget)
       ↓
INSERT INTO ai_model_usage
```

### Tracked Endpoints

| Endpoint | File | Purpose |
|----------|------|---------|
| `chat` | `chat/service.py` | Main QA response |
| `query_refinement` | `chat/query_refiner.py` | Query refinement before retrieval |
| `query_expansion` | `retrieval/expansion_service.py` | Multi-query expansion |
| `memory_extraction` | `chat/user_memory_service.py` | User memory extraction |
| `ragas_eval` | `analytics/ragas_evaluator.py` | RAGAS quality metrics (when enabled) |

### ai_model_usage Table

| Column | Source |
|--------|--------|
| `model_name` | `AIProxyBridge.model_name` |
| `prompt_tokens` | `provider.last_usage.prompt_tokens` |
| `completion_tokens` | `provider.last_usage.completion_tokens` |
| `endpoint` | Identifier (chat, query_refinement, etc.) |
| `cost_usd` | Estimated from `AI_INPUT_COST_PER_1M` + `AI_OUTPUT_COST_PER_1M` |
| `user_id`, `session_id`, `message_id` | Optional context for per-message attribution |

## Analytics Aggregation

```
ai_model_usage (all endpoint tokens)
       ↓
AnalyticsRepository: get_totals() → SUM tokens, cost by period
                     get_daily_stats() → GROUP BY date
                     get_recent_messages() → last N messages with per-message breakdown
       ↓
GET /analytics/stats → system-wide (admin) or personal (member)
```

Admin analytics now shows tokens from ALL AI calls (chat, refinement, expansion, compaction, memory, RAGAS), not just chat responses.

## Pricing Configuration

```bash
AI_INPUT_COST_PER_1M=0.0   # Set to actual provider rates
AI_OUTPUT_COST_PER_1M=0.0
```

## Analytics Endpoints

| Endpoint | Access | Returns |
|----------|--------|---------|
| `GET /analytics/stats` | Admin | System-wide token usage, cost, sessions |
| `GET /analytics/stats` | Member | Own usage only |
| `GET /analytics/messages` | Admin | Last 20 messages with per-message token details |
