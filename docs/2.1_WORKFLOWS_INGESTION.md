# 2.1 — Ingestion Workflow

Workflow này biến file upload thành dữ liệu có thể retrieval được trên dual-index mới.

## Tóm tắt

```text
Upload -> enqueue task -> parse -> canonical sections -> dual index -> finalize
```

## Luồng chính

1. `POST /v1/upload`
2. backend lưu file vào RustFS
3. tạo row trong `documents`
4. enqueue task ingest
5. worker đọc file và parse nội dung
6. normalize thành canonical `document_sections`
7. lưu `document_sections` vào PostgreSQL
8. build:
   - section index trong Qdrant
   - chunk index trong Qdrant
9. cập nhật document sang `ready`

## Thành phần chính

| Thành phần | File |
|---|---|
| Route upload | `app/modules/documents/router.py` |
| Task orchestration | `app/modules/documents/tasks.py` |
| Ingestion service | `app/modules/documents/ingestion/ingestion_service.py` |

## Canonical section graph

Vui lòng xem chi tiết Schema (các trường dữ liệu) của section graph tại mục **Canonical section graph** trong tài liệu [1_ARCHITECTURE.md](./1_ARCHITECTURE.md). Section graph này là source of truth trước khi index.

## Dual index & Payload rules

Pipeline Ingestion phải đảm bảo tạo payload hợp lệ và lưu vào 2 collection của Qdrant (sections và chunks).
Vui lòng xem chi tiết định nghĩa cấu trúc payload và vai trò của 2 collection này tại mục **Retrieval architecture** trong tài liệu [1_ARCHITECTURE.md](./1_ARCHITECTURE.md).

## LlamaIndex primitives đang dùng

- `SentenceWindowNodeParser`
- `HierarchicalNodeParser` (để ghi nhận cấu trúc chunking/hierarchy support)
- `VectorStoreIndex`

## Progress realtime

Progress tài liệu dùng **SSE với auto-reconnect**:

- backend cập nhật `status`, `status_stage`, `progress_percent`
- backend mở `GET /v1/documents/stream`
- frontend subscribe SSE để cập nhật gần realtime
- progress embedding/indexing được đẩy theo từng batch `section` và `chunk`, không chỉ theo mốc lớn

## Rechunk

Rechunk sẽ:

1. đọc lại artifact markdown đã có
2. xóa sections cũ + vectors cũ
3. parse lại canonical section graph
4. build lại dual index

Không cần upload lại file từ đầu.

## Parser metadata rule

- metadata passed into parser chunking must stay intentionally small
- `breadcrumb` and `breadcrumb_text` belong to retrieval payload, not the parser budget
- full metadata is attached after nodes are generated

## Invariants

- upload không block user chờ parse xong
- `tenant_id` phải đi xuyên suốt
- canonical section graph phải được lưu trước khi finalize
- rechunk phải xóa dữ liệu cũ trước khi ghi lại
- document status phải phản ánh đúng tiến độ thực tế
